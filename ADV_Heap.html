🏔️ Introduction to Heaps

A heap is a specialized tree-based data structure that satisfies the heap property. It's one of the most powerful data structures for efficiently managing prioritized data.

🎯 What is a Heap?

A heap is a complete binary tree where every node satisfies a specific ordering property:

DATA PROTOCOLS:

1. Max-Heap: Parent ≥ Children (Root is MAX)
2. Min-Heap: Parent ≤ Children (Root is MIN)
📊 Max-Heap Example
              100 (Root)
            /     \
          80       90
         /  \     /  \
       60   70   85   50

Analysis: Every parent is LARGER than its children.

⚡ Why Use Heaps?

ADVANTAGES:

• Access Min/Max: O(1)
• Insert/Delete: O(log n)
• Space: Efficient Array Storage

⚙️ Heap Operations

1. Heapify (Bubble Down)

Moves a node down until the heap property is restored.

def heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    
    if left < n and arr[left] > arr[largest]:
        largest = left
    
    if right < n and arr[right] > arr[largest]:
        largest = right
        
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

2. Array Indexing

FORMULAS (0-based):
Parent: (i - 1) // 2
Left Child: 2 * i + 1
Right Child: 2 * i + 2

🎯 Priority Queues

A priority queue serves elements based on priority rather than insertion order. Heaps are the standard way to implement them.

🐍 Python's heapq

import heapq

# Create min-heap
heap = []
heapq.heappush(heap, 10)
heapq.heappush(heap, 5)
heapq.heappush(heap, 20)

# Remove smallest
print(heapq.heappop(heap)) # Output: 5
