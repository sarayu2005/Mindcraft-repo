Protocol: Linked Lists

Concept & Structure

Unlike arrays, Linked Lists are not stored in contiguous memory. Elements are chained via pointers.

[ Data | Next ]
→
[ Data | Next ]
→
NULL

TACTICAL ADVANTAGE: Dynamic Size & Efficient Insertion/Deletion (O(1)). No memory shifting required.

Structure Types

Singly vs Doubly

1. Singly Linked List

Unidirectional. Forward movement only.

HEAD → [1] → [2] → NULL

2. Doubly Linked List

Bidirectional. Nodes have Next and Prev pointers.

NULL ← [1] ⇄ [2] → NULL

Technique: Floyd's Cycle

Fast & Slow Pointers

STRATEGY: Deploy two units at different speeds.

  • Slow Pointer: 1 step per turn.
  • Fast Pointer: 2 steps per turn.

APPLICATIONS:

  1. Find Middle: When Fast hits end, Slow is at midpoint.
  2. Cycle Detection: If loop exists, Fast "laps" Slow.

Protocol: Reversal

In-Place Modification

Before: 1 → 2 → 3 → NULL
After: 1 ← 2 ← 3 (New Head)

The 3-Pointer Strategy:

  • > prev: Tracks node behind.
  • > current: Node being processed.
  • > next_node: Saves forward path before breaking link.
